

### 基于SpringBoot+Vue+Ai的人力资源管理系统


1. 前端

   Vue、Axios、ElementUI、Vue-Router、Vuex、ECharts，pinia

2. 后端

   Spring Boot、JWT、MyBatis-Plus、MySQL、Hutool，Fastjson，OkHttp，Logback，Maven

系统设计
该系统主要是用来实现人力资源管理的系统，管理员为用户分配角色，不同的用户对应的权限不同，能使用的功能也不同，每个用户的基础功能就是访问主页，请假，查看考勤状况等等。该系统的主要流程是：注册，登录，进入主页，修改密码，切换头像，AI对话，请假，查看考勤记录等等功能，退出。
功能结构

本系统主要分五个模块，分别是系统管理和权限管理、薪资管理、考勤管理，AI对话。系统管理主要用于日常事务管理管理，权限管理，用于控制员工的访问权限，薪资管理主要是对员工的五险一金以及社保数据的修改和添加，考勤管理主要是对员工的日常打卡进行记录和统计，AI对话主要是依赖于大模型实现的会话框，帮助用户办公，解答，提升工作效率。


通过导入员工的考勤记录表，系统就会根据员工所在部门规定的上班时间进行比对，判断员工是否迟到、早退、旷工。（这是数据导入的模板，需要按照此格式填写数据。系统只会读取上下班时间的时间部分，系统以考勤日期作为打卡日期）。
 
另外，介绍一下考勤规制：

* 若考勤日期对应的是周末，则不判读员工考勤状态，默认为休假。
* 若考勤日期对应的是员工请假的日期，则员工考勤状态设为休假。
* 若员工的四个打卡时间，只要有一个为空，则员工考勤状态设为旷工。
* 若员工既迟到又早退，则员工考勤状态设为旷工。
* 若员工迟到，则员工考勤状态设为迟到。
* 若员工早退，则员工考勤状态设为早退。
* 若不是以上任何情况，则员工考勤状态设为正常。

项目配置

1. 前端
hrm系统：
   在`.env`文件中
   ```xml
   # 修改端口号，此端口号与在application.yml中配置的端口号相同
   VUE_APP_PORT = 8888
   ```

AiChat系统：
   npm install
   npm run dev
   前端开启端口：应该为：3001
2. 后端
hrm系统：

   在`application.yml`中进行相关的配置

   ```xml
   # 这是后端项目的运行端口，可自行修改
   server:
     port: 8888
   
   修改数据源，若MySql数据库版本较低，此处应该是com.mysql.jdbc.Driver
   driver-class-name: com.mysql.cj.jdbc.Driver
   
   修改数据库信息，确保与本机的数据库信息相符（密码，账号，端口，数据库名称等）

   # 因为此项目涉及到了文件上传与下载，此路径用来存储上传的文件。
   file:
     upload:
       path: E:/project/idea/hrm/file/ # 修改为自己的路径
   ```

AiChat系统：
修改application配置文件中内容，确保与自身环境一致
mvn install 重新安装一次maven依赖，确保不会冲突
      配置端口，以防出现端口冲突导致启动失败
      配置获得的讯飞星火的APIPASSWD，并配置API，调用讯飞星火大模型
      配置ollama的环境，本地部署deepseekR1-1.5b模型，调用端口，使用大模型
      使用postman测试，确保两个大模型可以使用
      启动后端，应正常开启，端口应该在6060
